# Git ガイドライン

## コミットメッセージ

[Conventional Commits](https://www.conventionalcommits.org/) に従う。

```
<prefix>: <要約（日本語）>
```

### prefix 一覧

| prefix | 用途 |
|---|---|
| `feat:` | 新機能 |
| `fix:` | バグ修正 |
| `refactor:` | 機能変更を伴わないコード改善 |
| `docs:` | ドキュメントのみの変更 |
| `style:` | フォーマット変更（動作に影響しない） |
| `chore:` | ビルド設定、依存関係の更新など |

### ルール

- 要約は日本語で簡潔に書く
- body が必要な場合は空行を挟んで詳細を書く
- 1コミット1関心事。変更が複数の関心事にまたがる場合は分割する

### 例

```
feat: アンケート作成 API を追加

Survey モデルに対する CRUD エンドポイントを実装。
```

```
fix: 回答送信時のバリデーションエラーを修正
```

## ブランチ

| パターン | 用途 |
|---|---|
| `main` | 本番ブランチ |
| `feat/<name>` | 機能開発 |
| `fix/<name>` | バグ修正 |
