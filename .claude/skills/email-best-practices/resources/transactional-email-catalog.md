# トランザクショナルメールカタログ

カテゴリ別のトランザクショナルメール一覧と、アプリの種類ごとの推奨メール組み合わせ。

## このガイドを使うとき

- アプリに必要なトランザクショナルメールを計画するとき
- アプリの種類に適したメールを選択するとき
- 各メールの種類に含めるべき内容を理解するとき
- トランザクショナルメール機能を実装するとき

## アプリの種類別メール組み合わせ

構築するアプリに応じて、以下を出発点として使用する。

### 認証重視のアプリ

ユーザーアカウントとセキュリティがコアとなるアプリ（ログインシステム、ID プロバイダ、アカウント管理）。

**必須:**
- メール認証
- パスワードリセット
- OTP / 2FA コード
- セキュリティアラート（新しいデバイス、パスワード変更）
- アカウント更新通知

**オプション:**
- ウェルカムメール（プロモーションを含めないこと）
- アカウント削除確認

### ニュースレター / コンテンツプラットフォーム

コンテンツ配信とサブスクリプションに特化したアプリ。

**必須:**
- メール認証
- パスワードリセット
- ウェルカムメール（プロモーションを含めないこと）
- 購読確認

**オプション:**
- OTP / 2FA コード
- アカウント更新通知

### EC / マーケットプレイス

ユーザーが商品やサービスを購入するアプリ。

**必須:**
- メール認証
- パスワードリセット
- ウェルカムメール（プロモーションを含めないこと）
- 注文確認
- 配送通知
- 請求書 / 領収書
- 決済失敗通知

**オプション:**
- OTP / 2FA コード
- セキュリティアラート
- 購読確認（定期注文向け）

### SaaS / サブスクリプションサービス

有料サブスクリプション階層と継続的な課金があるアプリ。

**必須:**
- メール認証
- パスワードリセット
- ウェルカムメール（プロモーションを含めないこと）
- OTP / 2FA コード
- セキュリティアラート
- 購読確認
- 購読更新通知
- 決済失敗通知
- 請求書 / 領収書

**オプション:**
- アカウント更新通知
- 機能変更通知（破壊的変更の場合）

### 金融 / フィンテックアプリ

お金、決済、機密性の高い金融データを扱うアプリ。

**必須:**
- メール認証
- パスワードリセット
- OTP / 2FA コード（機密操作に必須）
- セキュリティアラート（すべての種類）
- アカウント更新通知
- 取引確認
- 請求書 / 領収書
- 決済失敗通知

**オプション:**
- ウェルカムメール（プロモーションを含めないこと）
- コンプライアンス通知

### SNS / コミュニティプラットフォーム

ユーザー間のインタラクションとコミュニティ機能に特化したアプリ。

**必須:**
- メール認証
- パスワードリセット
- ウェルカムメール（プロモーションを含めないこと）
- セキュリティアラート

**オプション:**
- OTP / 2FA コード
- アカウント更新通知
- アクティビティ通知（メンション、返信）

### 開発者ツール / API プラットフォーム

API アクセスとインテグレーションを持つ開発者向けアプリ。

**必須:**
- メール認証
- パスワードリセット
- OTP / 2FA コード
- セキュリティアラート
- API キー通知（作成、有効期限）
- 購読確認
- 決済失敗通知

**オプション:**
- ウェルカムメール（プロモーションを含めないこと）
- 使用量アラート（制限に近づいた場合）
- 機能変更通知

### ヘルスケア / HIPAA 準拠アプリ

保護対象医療情報（PHI）を扱うアプリ。

**必須:**
- メール認証
- パスワードリセット
- OTP / 2FA コード（必須）
- セキュリティアラート（すべての種類、詳細に）
- アカウント更新通知
- 予約確認

**オプション:**
- ウェルカムメール（プロモーションを含めないこと）
- コンプライアンス通知

**注意:** ヘルスケアアプリには厳格な要件がある。メールには最小限の PHI のみを含め、機密情報にはセキュアなポータルへのリンクを使用すること。

## 完全メールカタログ

### 認証とセキュリティ

#### メール認証 / アカウント認証

**送信タイミング:** ユーザーがサインアップした直後、またはメールアドレスを変更した時。

**目的:** メールアドレスがユーザー本人のものであることを確認。

**含めるべき内容:**
- 明確な認証リンクまたはコード
- 有効期限（通常 24〜48 時間）
- 手順の説明
- 誤ってリンクをクリックした場合のセキュリティ通知

**ベストプラクティス:**
- 即座に送信（数秒以内）
- 有効期限の明記
- 再送オプションの提供
- 問題がある場合のサポートリンク

#### OTP / 2FA コード

**送信タイミング:** ユーザーが二要素認証コードを要求した時。

**目的:** 時間制限のある認証コードの提供。

**含めるべき内容:**
- OTP コード（明確に表示）
- 有効期限（通常 5〜10 分）
- セキュリティ警告
- リクエストしていない場合の対処方法

**ベストプラクティス:**
- 即座に送信
- コードを大きく読みやすく表示
- 有効期限を目立つように記載
- コードの共有に関する警告
- 「心当たりがない場合」リンクの提供

#### パスワードリセット

**送信タイミング:** ユーザーがパスワードリセットを要求した時。

**目的:** 忘れたパスワードを安全にリセットできるようにする。

**含めるべき内容:**
- リセットリンク（トークン付き）
- 有効期限（通常 1 時間）
- セキュリティ警告
- リクエストしていない場合の説明

**ベストプラクティス:**
- 即座に送信
- リンクは短時間で失効（1 時間）
- 可能であれば IP アドレスと場所を含める
- 「心当たりがない場合」リンクの提供
- 旧パスワードを含めない

#### セキュリティアラート

**送信タイミング:** セキュリティ関連のイベント発生時（新しいデバイスからのログイン、パスワード変更など）。

**目的:** アカウントのセキュリティイベントをユーザーに通知。

**含めるべき内容:**
- 発生した事象の明確な説明
- 発生日時
- 場所/IP（可能な場合）
- 不審な場合の対処方法
- セキュリティ設定へのリンク

**ベストプラクティス:**
- 即座に送信
- 明確で具体的に記述
- 実行可能な手順を含める
- 不審な操作を報告する方法を提供

### アカウント管理

#### ウェルカムメール

**送信タイミング:** アカウント作成とメール認証の完了直後。

**目的:** 新しいユーザーを歓迎し、次のステップに案内する（プロモーションを含めないこと）。

**含めるべき内容:**
- ウェルカムメッセージ
- 主要機能または次のステップ
- 重要なリソースへのリンク
- サポート連絡先

**ベストプラクティス:**
- メール認証後に送信
- 焦点を絞ってアクション可能に
- 情報過多にしない
- 今後のメールについての期待値を設定

#### アカウント更新通知

**送信タイミング:** ユーザーがアカウント設定を変更した時（メール、パスワード、プロフィールなど）。

**目的:** アカウント変更の確認とセキュリティ通知の提供。

**含めるべき内容:**
- 変更された内容
- 変更日時
- 不正な操作の場合の対処方法
- アカウント設定へのリンク

**ベストプラクティス:**
- 変更直後に送信
- 変更内容を具体的に記述
- セキュリティ通知を含める
- 必要に応じて元に戻す方法を提供

### EC とトランザクション

#### 注文確認

**送信タイミング:** 注文の直後。

**目的:** 注文内容の確認とレシートの提供。

**含めるべき内容:**
- 注文番号
- 注文商品と数量
- 価格の内訳
- 配送先住所
- 配送予定日
- 注文追跡リンク（利用可能な場合）

**ベストプラクティス:**
- 注文から数分以内に送信
- すべての注文詳細を含める
- 印刷・保存しやすくする
- カスタマーサービスの連絡先を提供

#### 配送通知

**送信タイミング:** 注文が発送された時、追跡更新とともに。

**目的:** 注文の発送をユーザーに通知し、追跡情報を提供。

**含めるべき内容:**
- 注文番号
- 追跡番号
- 配送業者情報
- 配送予定日
- 追跡リンク
- 配送先住所の確認

**ベストプラクティス:**
- 発送時に送信
- 追跡番号を目立つように表示
- 配送業者の追跡リンクを提供
- 主要な追跡マイルストーンで更新

#### 請求書と領収書

**送信タイミング:** 決済処理後。

**目的:** 決済確認と領収書の提供。

**含めるべき内容:**
- 請求書/領収書番号
- 決済金額
- 決済方法
- 購入した商品/サービス
- 決済日
- ダウンロード可能な PDF（該当する場合）

**ベストプラクティス:**
- 決済直後に送信
- すべての決済詳細を含める
- ダウンロード/保存しやすくする
- 該当する場合は税金情報を含める

### サブスクリプションと課金

#### 購読確認

**送信タイミング:** ユーザーが購読した時、または購読を変更した時。

**目的:** 購読内容と課金情報の確認。

**含めるべき内容:**
- サブスクリプションプランの詳細
- 課金金額と頻度
- 次回の課金日
- 決済方法
- サブスクリプション管理へのリンク

**ベストプラクティス:**
- 購読直後に送信
- 課金条件を明確に記載
- 簡単なキャンセルオプションを提供
- サポート連絡先を含める

#### 購読更新通知

**送信タイミング:** サブスクリプション更新前（通常 3〜7 日前）。

**目的:** 更新と課金の予定をユーザーに通知。

**含めるべき内容:**
- 更新日
- 課金予定額
- 登録済みの決済方法
- 決済方法の更新リンク
- キャンセルリンク

**ベストプラクティス:**
- 十分な猶予を持って送信（3〜7 日前）
- 金額と日付を明確に記載
- 決済方法の更新を簡単にする
- キャンセルオプションを提供

#### 決済失敗通知

**送信タイミング:** サブスクリプションの決済が失敗した時。

**目的:** 決済の失敗をユーザーに通知し、解決手順を提供。

**含めるべき内容:**
- 発生した事象
- 失敗した金額
- 失敗の理由（判明している場合）
- 解決手順
- 決済方法の更新リンク
- 未解決の場合の影響

**ベストプラクティス:**
- 失敗直後に送信
- 影響を明確に記載
- 簡単な解決方法を提供
- サポート連絡先を含める

### 通知とアップデート

#### 機能変更通知（トランザクショナル）

**送信タイミング:** ユーザーが使用中の機能に大きな変更があった時。

**目的:** サービス利用に影響する変更をユーザーに通知。

**含めるべき内容:**
- 変更内容
- ユーザーへの影響
- 必要なアクション（ある場合）
- 詳細情報へのリンク

**ベストプラクティス:**
- 重要な変更のみ
- ユーザーへの影響に焦点を当てる
- 明確な次のステップを提供
- ドキュメントへのリンク

**注意:** 一般的な機能のお知らせはマーケティングメール。ユーザーが使用中の機能に直接影響する変更の場合のみ、トランザクショナルとして送信すること。
