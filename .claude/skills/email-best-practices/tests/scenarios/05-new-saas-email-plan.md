# シナリオ 5: 新規 SaaS メールインフラ計画

## プロンプト

```
あなたは AI コーディングアシスタントです。開発者から以下の質問があります:

「新しい SaaS アプリを構築していて、メールインフラ全体を計画する必要があります。知りたいのは: (1) どの種類のトランザクショナルメールを計画すべきか、(2) DNS 認証の設定方法、(3) 送信ドメインのウォームアップ方法、(4) 本番環境でのバウンス/苦情の処理方法、(5) 海外ユーザー向けのコンプライアンス要件。包括的な実装ロードマップを教えてください。」

具体的に記述すること: IP ウォーミングスケジュール（週ごとの日次送信量）、バウンス率の閾値、苦情率の閾値、DNS レコードのフォーマット、地域別の法的要件。
```
